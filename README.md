# LSTNN — Aligning Transformer Circuit Mechanisms to Neural Representations in Relational Reasoning
(readme largely generated by AI)
Public code repository for the research project **"Aligning transformer circuit mechanisms to neural representations in relational reasoning"** published in TBA.

This project investigates how Transformer neural network models solve the Latin Squares Task (LST) — a relational reasoning task — and compares their internal representations to fMRI data, as well as human behaviour. 
---

## Overview

The repository implements a pipeline including synthetic puzzle generation through neural network training to fMRI Representational Similarity Analysis (RSA):

1. **Generate synthetic LST puzzles** at three levels of relational complexity (Binary, Ternary, Quaternary)
2. **Train neural network models** (Transformer, LSTM, MLP) on the LST under various positional encoding schemes
3. **Process fMRI data** from human participants performing the same task (denoising, GLM, parcellation)
4. **Compute Representational Dissimilarity Matrices (RDMs)** from both model internal activations and brain activity
5. **Compare model RDMs to fMRI RDMs** via permutation testing to assess brain alignment
6. **Reproduce manuscript figures and high-level statistics** via Jupyter notebooks

## Setup

### Prerequisites

- Python 3.10+
- PyTorch (tested with 1.10+)
- fMRI pipeline requires access to BIDS-formatted neuroimaging data (not included)

## Brief Usage Documentation

### Train Models

**Transformer:**
```bash
python scripts/run_transformer.py \
    --seeds 2235 \
    --pe 1dpe \
    --lr 0.0001 \
    --layers 4 \
    --embedding-dim 160 \
    --attnheads 1 \
    --n_epochs 1000
```

**LSTM or FFN:**
```bash
python scripts/run_simple_models.py
```

### fMRI Processing

The fMRI GLM pipeline is orchestrated via Snakemake:

```bash
snakemake -s scripts/workflows/imaging/snakefile --configfile scripts/workflows/imaging/config.yml
```

### RSA Pipeline

```bash
python scripts/run_RSA.py
```

### Reproduce Manuscript Figures

Launch Jupyter and open notebooks in `scripts/`:

| Notebook | Manuscript Figure(s) |
|----------|---------------------|
| Analysis 1: Model Performance | Model accuracy comparisons |

| Analysis 2b: fMRI RSA | fMRI vs ANN RDMs |
| Analysis 3: Influence of PE | Positional encoding effects |
| Analysis 4a: Attention weights | Attention-reasoning complexity |
| Analysis 4b: Temperature | Attention temperature effects |
| Analysis 5: RSA Gradients | Gradient-based alignment |

## Models

### Transformer

The custom `Transformer` (`lstnn/transformer_main.py`) supports a wide range of **positional encoding (PE) variants**. These are described in another paper ["Learning interpretable positional encodings in transformers depends on initialization"](https://arxiv.org/abs/2406.08272). Of interest here is "2dpe" and "learn":

| Flag | PE Type | Description |
|------|---------|-------------|
| `2dpe` | Absolute 2D | Sinusoidal PE on height×width grid |
| `learn` | Learnable | Learned PE with init scale |

Architecture: embedding dim 160, 4 layers, 1 attention head, GELU activations, residual connections, layer norm, gradient checkpointing.

### LSTM

Stacked LSTM (`lstnn/lstm_main.py`): 4 layers, 160 hidden dim, optional bidirectionality.

### FFN (MLP)

Feedforward network (`lstnn/ffn_main.py`): Flatten → Linear(80,160) → ReLU → Linear(160,160) → ReLU → Linear(160,160) → ReLU → Linear(160,4) → LogSoftmax.

## Data

- **`processed_data/puzzle_data_original.csv`**: 108 original LST puzzles from the fMRI experiment; the validation puzzles for models
- **`processed_data/generated_puzzle_data_*.csv`**: Synthetic training puzzles at each complexity level
- **`processed_data/human_behaviour.csv`**: Human behavioural accuracy data
- **`processed_data/LST_data.mat`** / **`LST_step_data.mat`**: Original MATLAB trial data
- **FMRI data**: Raw BOLD data is not included; processed group-level RDMs are in `processed_data/fmri_rdms/`
- **Trained model checkpoints**: Available in `processed_data/ito_models/`

## Results & Reproducibility

All experiments use **15 random seeds** for robust results:
```
[2235, 6312, 6068, 9742, 8880, 2197, 669, 6256, 3309, 2541, 8643, 7785, 195, 6914, 29]
```

Statistical comparisons use permutation testing with FDR correction for multiple comparisons across brain regions.
